De Casteljau’s algorithm is an algorithm used for forming bezier curves via recursive linear interpolation. Between two points, we perform linear interpretation for all control points between the 2. We implemented this function by taking in an array of points which are represented in vector form. For each point in the input array, we calculate its linear interpolation with the formula ((1-t)*points[i]) + (t*points[i+1]) , where t is a number between 0 and 1 to evaluate the bezier curve at. This gives us our new point, which we store. We repeat this process until we iterate through all points except for the last point in the list (points.size() - 1). We then return the array of new points.

  1. Repeating the de Casteljau algorithm will give us a set of bezier curves. Repeating this process again for what we call 2D de Casteljau algorithm will create a bezier surface. This is done by taking all of the control points and evaluating them at de Casteljau’s 1D algorithm. Implementing this, meant taking an array of control points, where each element is a list of points. We then, evaluate each set of points using de Casteljau. The algorithm for de Casteljau is the same as in task 1, and returns an array of size one less than the original input array. To evaluate it at a 1D level, we repeat the de Casteljau algorithm until the returned array is size 1. Than final point returned will be our return for our 1D evaluation. Once we’ve gotten all the new points for the control points, we evaluate the new points via another 1D evaluation to get our final return point.

To implement area-weighted normals, we traverse the half edges to get to the vertices incident to our original vertex. To find the area of each face, we looped through half edges, ending the loop when we returned to the half edge with started with. This is done by calling, this->halfedge() which gives us a halfedge iterator, we’ll call iter. Inside the loop, we first check if the face is a boundary, in which case we skip that face. Otherwise, we calculated the position of each vertex as follows:

  • Orign: this->position
  • Point 1: *(iter->next()->vertex())
  • Point 2: *(iter->twin()->next()->next()->vertex())

We then use these points to calculate the area by getting the vectors AB = Point1.position - Origin.position and AC = Point2.position - Origin.Position and use these in the area calculation and normal calculation:

  • area = 0.5 * cross(AB, AC).norm()
  • normal = cross(AB, AC) * area * -1;

Finally we add this normal to another vector, the summedWeightedNormal. To get to the next face, we call Iter->twin()->next().  We then normalize the summedWeightedNormal, then return it.

We followed the given instructions for implementing loop subdivision. Starting by looping through all the vertices and calculating their new positions.

When looping, we repeat this process for every vertex. We traverse the vertices incident to the current vertex. This is done by an inner while loop, where we repeat a process similar to task 3, where we use a half edge iterator and keep track of the first half edge when we cal v-halfedge() where v is the current vertex. We then keep track of n, the number of incident vertices or v’s degree, and a vector sum which will sum all of the positions of the incident vertices. The traversal is just two calls. h->twin()->vertex()->position for the incident vertex’s position, then calling h->twin()->next() to get to the next incident vertex. Once we’ve caluclated te sum of positions and n, we check if n = 3, in which case u = 3/16 otherwise u = 3/(8 *n) as described by the spec. Finally, we store v’s new position as (1.0f - n * u) * v->position + (u * sum), then set v’s isNew property to false.

Now we move on to the new step, which is to compute the updated vertex positions and store them. We start by iterating through all of the meshe’s edges. For each edge e, we first get its halfedge, h = e->halfedge(). Using this halfedge we want to traverse the mesh edges so that we can find the vertex positions of a,b,c,d as per the spec. These are found via the following calls:

        a = h->vertex()->position;

b = h->twin()->vertex()->position;

c = h->next()->next()->vertex()->position;

d = h->twin()->next()->next()->vertex()->position;

Then, set e’s new position property to false, and we simply plug in these values into the given formula to find the current edge, e’s new position,:

        (3.0f/8.0f) * (a + b) + 1.0f/8.0f * (c + d)

The next step is to split the edges. Instead of directly iterating over the meshes edges since the list will be resized, we start with a value n which is equal to the number of meshes currently n = mesh.nEdges(). We also declare an edge iterator e, and just set it to e = mesh.edgesBegin(). Inside the loop, we check if e is an old edge, then we split that edge using the function from part 5. We set this new splitted edge’s position to the new position value we calculated and stored for e’s new position property. Then, we mark the new splitted edge as a new edge. Finally, we update e’s value calling e++, regardless if the e was new or not. 

The next step is to flip the edges connecting a new vertex to an old one. Notably, these are the BLUE edges in the spec. We being by iterating through all of the meshes edges. For each edge e, we check if that edge iks new. If so we check if the vertices connected by that edge are old and new or new and old. If that edge satisfies both of those properties, we flip the edge.

We move on to the last step, which is just to iterate through all of the meshes vertices, and update the vertices position with the new position we calculated in step one. We’re finally done, and can upsample our meshes.
