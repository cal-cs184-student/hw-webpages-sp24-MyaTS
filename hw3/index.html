Part 2

Constructing the BVH

Task 2.1: So we wrote a recursive function. When the node is a leaf node we want to set the start and end iterators and set the left and right child to null. Else we want to partition along the axis either x, y, or z depending on which extent on those axis is bigger. Then we calculate the split point by adding bbox.min and extent on the chosen axis, then divide by 2. Then we threw it into the partition function for convenience and made sure that the edge cases are covered for when the partition point lies directly either on the start or end iterator. Then we call the construct bvh recursively on the left and right node by passing in the split point calculated.

Intersecting the Bounding Box and Intersecting the BVH

Task 2.2: We calculated the intersection for bbox following the class discussion and updated the t0 and t1 values. A few errors that we had to debug were not calculating the min max for each point on the axis and reconfiguring to find the min of all the min values on the axis and find the max of all the max values on the axis. We also struggled with debugging in this project for a long time because we didn’t know that those final min and max values could be negative values. So all this time the function was returning false when it should be true.

Task 2.3: Our implementation is very similar to the pseudocode provided in lecture. The function has_intersection is a recursive function, if there is no bbox intersect it returns false, else it checks if it is a leaf node, if that’s the case, we want to loop over all the leaf node’s primitives and check if the primitive has intersection depending on what shape it is. If it does return true, else we call this function again recursively. The intersect function is also pretty much the same thing except if it is a leaf node, it calls intersect on the shape. With BVH intersection implemented, the result is so much faster because we test if each ray intersect the bounding box first before testing the intersection through the primitives. If it doesn’t even intersect the bounding box, all the primitives within it will be disregarded. This is obviously so much faster than just repetitively calculating and checking for ray-primitive intersection every single time for all primitives. This method means faster rendering time due to fewer tests being conducted as a result.

Direct Lighting with Uniform Hemisphere Sampling

Task 3.3: We mostly referenced to the equation in the lecture. We used Monte Carlo integration to figure out the estimate of the direct lighting on a point of intersection in a scene by uniformly sampling directions on a hemisphere. We created a for loop which uniformly samples a direction in the local coordinate space using get_sample(), then we converted it to world space, and created a sample ray on the given hit point. We set the appropriate min_t and max_t values. Then we check if the sample ray has intersection with the bvh->intersect method. If it does we did the calculation following the equation in the lecture and accumulate within L_out. We then returned the averaged L_out value over the num_samples.

Direct Lighting by Importance Sampling Lights

Task 3.4: This sampling makes the image better because we are using importance sampling (as described in the lecture slides) instead of uniform hemisphere sampling. This method sample light directly from the scene’s light sources which give better results. This function iterates over each light source in the scene. For each light, we determine whether it is delta light with requires only one sample, or it’s actually an area light which requires multiple samples using a for loop. For each sample, we generate a new ray from the hit point and do the same thing as what we did before like setting appropriate min_t and max_t values and calling bvh_intersect and using the equation provided in the lecture. We also did a similar thing for when sampling over a for loop, except this time we are averaging out L_out value over number of samples before returning it.

Conclusion: The difference in outcomes is the disparity in image quality and noise levels. Uniform sampling is grainier because of the association with high variance in it’s uniform sampling approach. Also many sample rays may have little to no lighting and that contributes to the increase in noise levels. For importance sampling, the result image is a lot more smoother and more seamless. This method reduces variance and noise by analyzing each light ray whether its a delta light or not and sampling accordingly (once or over a large area).

Part 4

Task 4 (summary): at_least_one_bounce_radiance computes the radiance that arrives at a point after at least one bounce which incorporates both direct and indirect illumination. First the function decides whether to continue tracing more bounces beyond the first one using Russian Roulette. Basically it uses probability to decide. The next step is very similar to what we did for part 3 functions, getting the components necessary such as bsdf, pdf, costheta, generating a new ray, and calling bvh intersect. Then we call the function recursively then return the normalized L_out accumulated. For indirect illumination this function is slightly different only because this time we are using depth recursively, like indirect light bounces starting from max_depth and decrementing until there is no more.
